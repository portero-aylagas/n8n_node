The most useful nodes in this lab were the trigger nodes, the AI model nodes, the Code node, the Notion node, and the HTTP Request node. I would choose nodes based on the task: trigger nodes to start execution, AI/model nodes to generate structured content, Code nodes to reshape or validate JSON when visual nodes are not enough, Split Out to turn arrays into item streams, and app/API nodes to write results into external systems. My top debugging tip is to inspect the input and output JSON after every node before changing configuration, because most n8n errors come from assuming a field exists in `$json` when the actual structure is nested differently or was transformed by a previous node.
